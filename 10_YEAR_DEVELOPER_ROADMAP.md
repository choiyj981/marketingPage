## 🎯 학습 접근법

### 핵심 원칙

1. **구현 우선, 이해는 나중에**
   - AI(Cursor) 도움을 받아 기능을 먼저 구현
   - 작동하는 코드를 보고 역공학으로 이해
   - "왜 이렇게 작동하는가?"를 나중에 질문

2. **DevOps 중심 학습**
   - 모든 기능을 배포 가능한 형태로 구현
   - CI/CD 파이프라인을 통해 자동화
   - 인프라를 코드로 관리(IaC)

3. **실전 프로젝트 기반**
   - 현재 마케팅 페이지 프로젝트를 확장
   - 각 단계마다 실제 사용 가능한 기능 추가
   - 포트폴리오로 활용 가능

## 📚 10주 커리큘럼 기반 10년 플랜

### 1주차: AI 개발 로직 & 환경 설정 (1개월)

**학습 내용:**
- Claude Code CLI 기반 프로젝트 세팅
- 프로덕션 코드와 테스트 코드를 포함한 초기 API 구현
- 구현 비용 하락에 따른 개발 흐름 변화 이해
- AI 시대에 요구되는 테스트 주도 개발(TDD)의 필요성과 가치 이해

**실무 과제:**
- 현재 프로젝트에 테스트 코드 추가
- CI/CD 파이프라인에 테스트 단계 추가
- TDD 방식으로 간단한 API 엔드포인트 구현

**10년 플랜 확장:**
- 1년차: 기본 TDD 습관화
- 2-3년차: 테스트 커버리지 80% 이상 유지
- 4-5년차: E2E 테스트 자동화
- 6-10년차: 테스트 전략 수립 및 팀 표준화

---

### 2~5주차: 아키텍처 설계 (4개월)

**학습 내용:**
- RESTful API 명세 설계 및 문서화
- 테스트 주도 개발(TDD)
- 단위 테스트 기반 점진적 구현
- 정규화/인덱싱 기반 DB 설계
- 통합 테스트 설계
- 트랜잭션 관리 및 Race Condition 대응 전략 수립

**실무 과제:**
- 요구사항 정의서를 기반으로 한 API 명세 및 데이터 구조 설계
- 테스트 코드 기반의 기능 구현 및 데이터 일관성 검증

**10년 플랜 확장:**
- 1년차: RESTful API 설계 및 문서화 (Swagger/OpenAPI)
- 2년차: GraphQL API 설계 및 구현
- 3년차: 마이크로서비스 아키텍처 설계
- 4년차: 이벤트 기반 아키텍처 설계
- 5년차: 도메인 주도 설계(DDD) 적용
- 6-7년차: 헥사고날 아키텍처, 클린 아키텍처 심화
- 8-10년차: 대규모 시스템 아키텍처 설계 및 리팩토링

---

**실무 과제:**

- Kafka 기반 비동기 메시징 시스템 설계

**10년 플랜 확장:**
- 1년차: Redis 기본 사용법 및 캐싱 전략
- 2년차: Redis 클러스터링 및 고가용성 구성
- 3년차: Kafka 기본 사용법 및 프로듀서/컨슈머 구현

- 5년차: Redis + Kafka 조합한 실시간 시스템 구축
- 6년차: Apache Flink를 활용한 스트림 처리
